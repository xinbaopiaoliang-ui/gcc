# v0.6.0 联调稳定版检查清单

适用对象：业务后台、客户端、控制面板、节点运维。

## 角色边界

- 业务后台维护游戏、进程、规则、节点、用户授权，生成客户端配置和节点 `route_policies`。
- 客户端只按业务后台配置识别本机进程和流量，不做全局代理默认转发。
- 节点只接收客户端通过 QUIC 发来的 `OPEN_TCP` / `OPEN_UDP`，并按 token + 本地策略校验后转发。
- 控制面板负责节点资产、策略版本、desired policy、一键部署/更新、节点上报、系统自检和接入诊断。

## 必须统一的 ID

| 字段 | 出现位置 | 要求 |
| --- | --- | --- |
| `node_id` | 业务后台节点、控制面板节点、节点配置 `node.id`、客户端节点配置 | 必须完全一致。 |
| `revision` / `config_revision` | 客户端配置、token claim、flow metadata、节点 `route_policies.revision` | 建议同一批配置使用同一个版本号。 |
| `game_id` | 业务后台游戏、客户端 metadata、token `game_ids`、节点策略 | 必须一致。 |
| `policy_id` | 业务后台策略、客户端 metadata、token `policy_ids`、节点策略 | 必须一致。 |
| `rule_id` | 业务后台规则、客户端 metadata、节点策略规则 | 必须一致。 |

## 控制面板上线检查

1. Go 后端启动：

```bash
curl http://127.0.0.1:<listen_port>/health
```

2. 业务后台自检：

```bash
curl http://127.0.0.1:<listen_port>/api/backend/system/check \
  -H "Authorization: Bearer <backend_api_key>"
```

要求：

- `summary.error = 0`。
- `database.schema` 为 `ok`。
- `backend_api_keys` 为 `ok`。
- `cors.origins` 包含前端站点，例如 `http://103.201.131.99:9788`。

3. 面板管理员登录后，在左侧进入“系统自检”，确认没有阻断性错误。

## 业务后台到控制面板

推荐顺序：

1. `POST /api/backend/nodes` 或 `PUT /api/backend/nodes/{node_id}` 同步节点。
2. `POST /api/backend/policy-revisions/validate` 校验策略。
3. `POST /api/backend/policy-revisions` 保存策略。
4. `POST /api/backend/nodes/{node_id}/desired-policy` 设置目标策略。
5. `GET /api/backend/nodes/{node_id}/sync-status` 等待 `policy_state=synced`。

业务后台不要调用节点 admin 口，也不要保存 SSH 密码明文。

## 客户端到节点

客户端启动配置必须至少包含：

- `revision`
- `device_id`
- `games[]`
- `nodes[].node_id`
- `nodes[].endpoint_host`
- `nodes[].endpoint_port`
- `nodes[].alpn`
- `nodes[].token`
- `nodes[].policies[]`

客户端打开 flow 时必须带 metadata：

```json
{
  "game_id": "steam",
  "policy_id": "steam-web-v1",
  "rule_id": "steam-store-tcp-443",
  "network": "tcp",
  "process_name": "steamwebhelper.exe",
  "client_config_revision": "20260618.1",
  "capture_mode": "process",
  "trace_id": "optional"
}
```

节点拒绝时，客户端必须优先使用 `ERROR.error_code` 排查，不要依赖错误文本。

## 节点接入诊断

控制面板节点列表点击“接入自检”。

重点看：

- 客户端入口是否完整：`endpoint_host:endpoint_port`。
- Admin 入口是否可达：`admin_host:admin_port`。
- `/health`、`/status`、`/panel/commands` 是否可访问。
- `node.id` 是否等于控制面板 `node_id`。
- `current_policy_revision` 是否等于 `desired_policy_revision`。

如果远程节点 `admin_host=127.0.0.1`，面板通常无法跨服务器直接探测，需要同机部署、反代或改成面板可访问地址。

## 通过标准

- 面板系统自检无 `error`。
- 业务后台保存策略前 validate 通过。
- 节点 sync-status 显示 `policy_state=synced`。
- 客户端能完成 QUIC `HELLO`、`AUTH`。
- TCP 和 UDP flow 都能在节点 metrics 中看到上下行字节。
- `/sessions` 能看到 `user_id`、`device_id`、`client_version`、`client_platform`、`game_id/policy_id/rule_id` 相关 flow 信息。
