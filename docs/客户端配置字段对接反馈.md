> 来源文件：`D:/Backup/Documents/xwechat_files/wxid_zbrt9m4fnnmf22_b016/msg/file/2026-06/游戏加速配置字段对接说明.md`
>
> 生成日期：2026-06-17
>
> 用途：给客户端和后台同事确认哪些字段需要继续保留，哪些字段需要转换后给节点，哪些字段当前节点不消费。

## 一、结论

客户端提供的配置字段整体可以使用，但不要把 `GM_GameZone` / `GM_AccNode` 的数据库字段原样理解为“全部下发给节点”。

当前节点实际消费的是三类数据：

1. 客户端连接节点时携带的短期 `token`。
2. 客户端每次 `OPEN_TCP` / `OPEN_UDP` 发给节点的 `target_host`、`target_port` 和 `metadata`。
3. 业务后台同步到节点本地的 `route_policies` 策略快照。

客户端本机进程识别、hosts、智能 DNS、清 hosts、伪 ping、三网 IP 选择、线路质量筛选等逻辑，属于客户端或业务后台逻辑，不是节点在连接时直接消费的字段。

## 二、职责边界

| 模块 | 负责内容 |
| --- | --- |
| 业务后台 | 维护游戏、区服、节点、用户授权；生成客户端配置；生成节点策略；签发短期 token |
| 客户端 | 按进程、域名、IP、协议、端口命中规则；选择节点；连接 QUIC；发起 `OPEN_TCP` / `OPEN_UDP` |
| 节点 | 校验 token；校验 `game_id` / `policy_id` / `rule_id` / `network` / `target_host` / `target_port`；通过后转发 TCP/UDP |

节点不会识别客户端本机进程，也不会执行 hosts、DNS、清理 hosts、伪 ping、启动命令等客户端本机操作。

## 三、客户端需要保留的字段

这些字段客户端需要消费，但不要直接全部发给节点。

| 客户端原字段 | 是否需要 | 用途 | 给节点方式 |
| --- | --- | --- | --- |
| `GM_GameID` | 需要 | 标识游戏 | 后台转换成 `game_id`，写入 token、客户端配置和节点策略 |
| `GM_GameZoneID` | 需要 | 标识区服/策略 | 后台转换成 `policy_id`，写入 token、客户端配置和节点策略 |
| `ZoneName` | 需要 | 客户端展示 | 不发节点 |
| `Version` | 需要 | 客户端判断配置是否更新 | 后台建议生成统一 `revision`，客户端放入 `metadata.client_config_revision` |
| `ProcessRules.ProcessName` | 需要 | 客户端本机进程匹配 | 只把命中的 `process_name` 放入 metadata，节点仅做日志 |
| `ProcessRules.Sign` | 可选 | 客户端本机进程校验 | 不发节点 |
| `ProcessRules.Md5` | 可选 | 客户端本机进程校验 | 不发节点 |
| `DomainRules` | 需要 | 客户端命中 TCP/UDP 域名规则 | 后台转换成节点 `route_policies.rules` |
| `IpRules` | 需要 | 客户端命中 TCP/UDP IP/CIDR 规则 | 后台转换成节点 `route_policies.rules` |
| `Hosts` | 视客户端能力 | 写系统 hosts | 不发节点 |
| `SmartDns` | 视客户端能力 | 客户端 DNS 处理 | 不发节点 |
| `CmdExec` | 视客户端能力 | 加速前本机命令 | 不发节点 |
| `SkipMainlandIP` | 视客户端能力 | 客户端本地跳过内地 IP | 不发节点 |
| `ClearHosts` | 视客户端能力 | 客户端本机清 hosts | 不发节点 |
| `FakePingValue` | 视客户端能力 | 客户端本机展示或 hook | 不发节点 |

## 四、节点字段映射

### 4.1 节点连接配置

客户端最终连接节点只需要这些字段：

```json
{
  "node_id": "node-test-01",
  "endpoint_host": "195.245.242.9",
  "endpoint_port": 5555,
  "alpn": "gaccel/1",
  "sni": "",
  "insecure": false,
  "token": "short-lived-jwt",
  "policies": ["policy-id-1"]
}
```

`GM_AccNode` 字段建议这样处理：

| 客户端原字段 | 是否需要 | 建议映射 |
| --- | --- | --- |
| `GM_AccNodeID` | 需要 | 转成 `node_id`，必须用字符串 |
| `NodeName` | 需要 | 客户端展示，不发节点 |
| `ServerIP` | 需要 | `endpoint_host` |
| `ServerPort` | 需要 | `endpoint_port` |
| `DisableQuic` | 需要过滤 | 当前节点协议就是 QUIC；如果为 true，当前客户端不应选择该节点 |
| `Area` | 需要 | 客户端筛选/展示，不发节点 |
| `BandwidthQuality` | 需要 | 客户端或后台筛选节点，不发节点 |
| `IsSupportIpv6` | 可选 | 客户端选择 IPv4/IPv6 入口，不发节点 |
| `TelecomIP` / `MobileIP` / `UnicomIP` | 可选 | 客户端或后台按运营商选择最终 `endpoint_host` |
| `RelayServerIP` / `RelayServerPort` | 暂不接入节点协议 | 如需中继，应由后台/客户端先选出最终入口；当前节点协议无 relay 字段 |
| `IsLocalIP` | 可选 | 客户端或后台筛选，不发节点 |
| `Tag` | 可选 | 客户端或后台筛选，不发节点 |
| `Status` | 需要 | 后台过滤禁用节点，不下发禁用项 |
| `Sort` | 需要 | 客户端排序，不发节点 |
| `Remark` / `CDate` / `UDate` | 不需要 | 不下发客户端或仅后台管理 |

### 4.2 每次 OPEN 发给节点的字段

客户端命中规则后，发给节点的是这个格式，不是把整份区服配置发给节点：

```json
{
  "type": "OPEN_TCP",
  "target_host": "store.steampowered.com",
  "target_port": 443,
  "metadata": {
    "game_id": "game-id",
    "policy_id": "policy-id",
    "rule_id": "rule-id",
    "network": "tcp",
    "process_name": "game.exe",
    "process_path_hash": "optional-sha256",
    "client_config_revision": "20260617.1",
    "capture_mode": "process",
    "trace_id": "optional-trace-id"
  }
}
```

必填字段：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `target_host` | 是 | 客户端实际访问的域名或 IP |
| `target_port` | 是 | 客户端实际访问端口 |
| `metadata.game_id` | 是 | 必须和 token、节点策略一致 |
| `metadata.policy_id` | 是 | 必须和 token、节点策略一致 |
| `metadata.rule_id` | 是 | 必须属于对应 `policy_id` |
| `metadata.network` | 是 | `tcp` 或 `udp`，必须和 OPEN 类型一致 |
| `metadata.client_config_revision` | 是 | 必须和 token 的 `config_revision` 一致，除非 token 未写该字段 |
| `metadata.process_name` | 建议 | 仅用于日志和排查，节点不靠它授权 |
| `metadata.process_path_hash` | 可选 | 日志/排查 |
| `metadata.capture_mode` | 建议 | 如 `process`、`wfp`、`windivert`、`tun` |
| `metadata.trace_id` | 可选 | 客户端生成，便于链路排查 |

## 五、后台需要转换成节点策略的字段

节点策略只需要安全校验所需字段：

```json
{
  "revision": "20260617.1",
  "policies": [
    {
      "policy_id": "policy-id",
      "game_id": "game-id",
      "allow_tcp": true,
      "allow_udp": true,
      "rules": [
        {
          "rule_id": "rule-id",
          "network": "tcp",
          "target_type": "domain_suffix",
          "target_value": ".steampowered.com",
          "port_start": 443,
          "port_end": 443,
          "action": "quic_relay",
          "priority": 10,
          "enabled": true
        }
      ]
    }
  ]
}
```

### 5.1 DomainRules 转换规则

| 客户端原字段 | 节点字段 | 转换说明 |
| --- | --- | --- |
| `Protocol` | `network` | `tcp` / `udp` / `any`；建议拆成 tcp、udp 两条规则，日志更清楚 |
| `Domain` | `target_type` + `target_value` | 精确域名用 `domain`；`*.example.com` 转成 `domain_suffix` + `.example.com` |
| `Port` | `port_start` / `port_end` | 逗号和范围需要拆成多条规则 |
| `IsAcc=true` | `action=quic_relay` | 命中后走节点 |
| `IsAcc=false` | 客户端本地 direct/block | 不建议发起 OPEN；如写入节点策略，节点会拒绝 relay |

注意：当前节点策略不支持任意中间通配，例如 `api*.example.com`。这种规则客户端可以本地匹配，但同步给节点时需要转换成精确域名、后缀域名，或另行扩展节点匹配能力。

### 5.2 IpRules 转换规则

| 客户端原字段 | 节点字段 | 转换说明 |
| --- | --- | --- |
| `Protocol` | `network` | `tcp` / `udp` / `any` |
| `IpCidr` | `target_type=cidr` + `target_value` | CIDR 可直接使用 |
| `Port` | `port_start` / `port_end` | 逗号和范围需要拆成多条规则 |
| `IsAcc=true` | `action=quic_relay` | 命中后走节点 |
| `IsAcc=false` | 客户端本地 direct/block | 不建议发起 OPEN |

重要：当前节点校验端口范围是 `1-65535`。客户端文档里的 `"0-65535"` 需要在后台转换为 `1-65535`，否则节点策略校验不会通过。

## 六、token 需要的信息

客户端拿到的是签发后的短期 JWT token，不需要也不能拿 `hmac_secret`。

建议 token claims：

```json
{
  "sub": "user-id",
  "user_id": "user-id",
  "device_id": "device-id",
  "exp": 1781593681,
  "nbf": 1781590076,
  "iat": 1781590081,
  "max_connections": 2,
  "rate_limit_mbps": 50,
  "allow_tcp": true,
  "allow_udp": true,
  "game_ids": ["game-id"],
  "policy_ids": ["policy-id"],
  "config_revision": "20260617.1"
}
```

客户端只保存和使用 `token`。`hmac_secret`、`token-api api_key`、`panel.command_secret`、TLS 私钥都不能下发客户端。

## 七、当前不需要给节点的字段

这些字段可以继续保留在后台或客户端配置里，但节点当前不消费：

| 字段 | 说明 |
| --- | --- |
| `GM_Game.GameName` / 图片 / 封面 / 背景图 | 客户端 UI 展示 |
| `GM_Game.StartUrl` | 客户端启动游戏 |
| `GM_GameTypeRel` / `GM_GamePlatformRel` | 客户端分类、平台筛选 |
| `ZoneName` / `Sort` / `Remark` | 客户端展示或后台管理 |
| `Hosts` / `ClearHosts` | 客户端本机 hosts 操作 |
| `SmartDns` | 客户端本机 DNS 处理 |
| `CmdExec` | 客户端本机命令 |
| `FakePingValue` | 客户端本机展示或 hook |
| `SkipMainlandIP` | 客户端本机规则判断或后台预处理 |
| `BandwidthQuality` | 客户端或后台选择节点，不参与节点单次 OPEN 校验 |
| `TelecomIP` / `MobileIP` / `UnicomIP` | 客户端或后台先选最终入口，节点不关心来源运营商 |
| `RelayServerIP` / `RelayServerPort` | 当前节点协议无 relay 字段 |
| `CDate` / `UDate` | 后台管理字段 |

## 九、需要确认的点

1. `GM_GameID` / `GM_GameZoneID` 是否直接作为 `game_id` / `policy_id` 使用。如果使用雪花 ID，必须全程字符串。
2. `Version` 是每个区服一个版本，还是后台会生成全局 `revision`。节点 token 和 metadata 需要使用同一套 `config_revision`。
3. `Port` 里的 `0-65535` 需要改成 `1-65535` 后再同步节点策略。
4. `Domain` 如果存在 `api*.example.com` 这种中间通配，当前节点策略无法精确表达，需要后台转换或节点扩展。
5. `DisableQuic=true` 的节点当前不能用于现有 QUIC 节点协议，客户端应过滤。
6. 三网 IP 是后台选最终入口，还是客户端按本机运营商选择。选完后只把最终 IP 放到 `endpoint_host`。
7. 中继字段当前未进入节点协议。如要支持中继，需要单独设计客户端到中继、或中继到节点的协议。
8. `IsAcc=false` 必须在客户端本地当作排除/direct/block 处理，不要命中后继续发给节点。
